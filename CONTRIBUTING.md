# Contributing

Contributions should keep this repository focused on UAVFF3D fine-tuning and evaluation.

Before opening a pull request:

1. Keep comments and documentation in English.
2. Avoid adding unrelated MapAnything demo, profiling, calibration, or non-UAV benchmark code.
3. Keep bash launchers under `bash_scripts/benchmark/uav_dense_n_view/` or `bash_scripts/train/uav_finetuning/` unless a new directory is clearly required by the UAVFF3D paper experiments.
4. Document any new third-party dependency and its license in `THIRD_PARTY.md`.
5. Run Python syntax checks on changed project files.
